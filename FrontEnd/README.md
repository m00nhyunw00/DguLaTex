# 🎨 Front-End (React + Vite)

사용자가 보는 화면과 데이터를 연결하는 프론트엔드입니다.

<br>

### 📁 폴더 구조 및 아키텍처 역할 분담

우리 프로젝트는 비즈니스 로직과 UI가 분리된 레이어드 아키텍처(Layered Architecture)를 따르며, 각 폴더는 다음과 같은 명확한 역할을 가집니다.

| 폴더명                 |           역할            | 설명                                   |
|:--------------------|:-----------------------:|:-------------------------------------| 
| **src/api/**        |      **Protocol**       | 백엔드 통신을 위한 기본 설정 (Axios Instance 등)  |
| **src/services/**   |    **Model (Data)**     | 서버(DB)에 데이터를 요청하는 순수 통신 로직           |
| **src/hooks/**      |     **Controller**      | 비즈니스 로직 처리 및 상태 관리 (Custom Hooks)    | 
| **src/pages/**      |    **View (Layout)**    | 훅과 UI를 조립하여 구성하는 독립적인 화면 단위          | 
| **src/components/** |      **View (UI)**      | 재사용 가능한 순수 디자인 부품 (Atomic Components) |
| **src/testdata/**   |      **Mock Data**      | API 연결 전 사용하는 임시 테스트 데이터 모음          |
| **src/assets/**     |       **Static**        | 로고, 이미지, 아이콘 등 정적 자원 보관              | 
| **pakage.json**     | **Manifest** | 프로젝트 정보, 라이브러리 의존성 및 실행 스크립트 관리 |
| **.env**            | **Environment Variables**   | 서버 주소 등 보안 및 환경 설정 변수 관리 (공개 공유 X) |

<br>

### 🔄 데이터 흐름 (Data Flow)

모든 데이터는 아래와 같은 단방향 흐름으로 처리되어 유지보수성을 극대화합니다.

1. **Service (데이터 획득)**: `src/services/`에서 API (`src/api/`) 를 호출하여 서버나 Mock 데이터로부터 순수 JSON 데이터를 가져옴
2. **Hook (로직 및 가공)**: `src/hooks/`에서 데이터를 상태 (`useState`) 에 저장하고, 수정/삭제/재귀 탐색 등 복잡한 비즈니스 로직을 수행함
3. **Page (조립)**: `src/pages/`에서 필요한 훅을 호출하고 가공된 데이터와 핸들러를 UI에 주입함~~~~
4. **Component (렌더링)**: `src/components/`는 전달받은 데이터를 화면에 출력하며, 데이터의 출처는 알지 못함

<br>

### 🔄 데이터 흐름 시나리오

백엔드 서버 및 데이터베이스(DB)와 통신할 때의 실제 로직 실행 순서입니다.
데이터의 요청부터 화면 출력까지 모든 과정은 비동기(Asynchronous)로 처리됩니다.


1. **페이지 접속 (`DashboardPage.jsx`)**
    - 사용자가 대시보드 화면에 들어오면 해당 페이지 컴포넌트가 실행됨
2. **비즈니스 로직 가동 (`useDashboard.js`)**
    - 페이지 내부에서 훅을 호출하여 데이터를 가져올 준비를 마침
3. **데이터 요청 명령 (`ProjectService.js`)**
    - 훅이 서비스 파일의 함수를 실행하여 백엔드 서버에 데이터 요청을 보냄
4. **DB 조회 및 서버 응답 (Backend API)**
    - 서버가 데이터베이스(DB)에서 프로젝트 목록을 찾아 JSON 형태로 프론트엔드에 전달함
5. **데이터 상태 저장 (`useDashboard.js`)**
    - 서비스로부터 전달받은 데이터를 훅 내부 상태(`projects`)에 저장함
6. **화면 렌더링 (`DashboardUI.jsx`)**
    - 저장된 데이터를 UI 컴포넌트로 전달하여 사용자 화면에 최종 출력함
- DB 연결 전에는 훅에서 테스트 데이터 (`projectData.js`) 를 직접 훅 내부 상태(`projects`)에 저장함

### 🏗️ 컴포넌트 설계 원칙

* **3단 레이아웃 분리**: 에디터 및 히스토리와 같은 복잡한 화면은 `Section` 단위로 부품을 쪼개어 독립적으로 관리합니다.
* **비즈니스 로직 캡슐화**: 모든 데이터 통신과 상태 변경 로직은 `hooks/` 내부에 위치하며, UI는 오직 "보여주는 법" 에만 집중합니다.
* **일관된 컬러 디자인**: `Common.css`의 `--dgu-orange` 변수를 사용하여 동국대학교 전용 프로그램이라는 정체성을 전역적으로 유지합니다.

### 🚀 실행 방법
1. `cd FrontEnd`
2. `npm install` (최초 1회)
3. `npm run dev`

### ⚠️ 주의사항
- **환경 변수**: 백엔드 서버 주소는 `.env` 파일의 `VITE_API_URL`을 참조하세요.
- **코드 분리**: 로직(Controller)은 `hooks`에, 통신(Model)은 `services`에 작성하여 페이지 컴포넌트의 비대화를 방지합니다.
- **주석 규격**: 모든 함수와 컴포넌트 상단에는 역할과 기능을 명시하는 헤더 주석을 작성하며, 문장 끝에 마침표(`.`)를 찍지 않는 규칙을 권장합니다.
- **환경 변수**: .env 파일은 GitHub에 포함되어있지 않으니, 따로 작성하거나 제공을 요청하세요.
